Mapping

The mapping between UML and SQL is done according to the following:

SQLUML
TableClass (stereotype <<Table>> is optional)
ColumnAttribute
RelationshipAssociation
Primary/Foreign key Attribute of special stereotype (<<PK>>, <<FK>>)
(Unique) index Class of stereotype <<Index>> or <<Unique>> with dependency to corresponding table
Attribute propertiesAttribute properties, special stereotypes (see below)

Modelling Foreign Keys

For modelling foreign keys it is not enough to just let an attribute be of the stereotype <<FK>>. It must also be specified what table and what column this foreign key refers to. This might be done using tagged values whereas source column and table should be used.

Attribute mapping

Special SQL-attribute properties will be mapped this way:

SQLUML
NULL / NOT NULLIf an attribute should be NOT NULL, let the attribute be of stereotype <<NOT NULL>>. The stereotype <<NULL>> is not needed on every attribute that is nullable, so the attribute is nullable if it is not <<NOT NULL>>.
autoincstereotype <<AutoInc>>

The 'default value' and the 'enumeration' attribute properties are supported directly.
