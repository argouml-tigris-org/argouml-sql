Mapping

The mapping between UML and SQL is done according to the following:

SQLUML
TableClass (stereotype <<Table>>)
ColumnAttribute
RelationshipAssociation
Primary/Foreign key Attribute of special stereotype (<<PK>>, <<FK>>)
(Unique) index Method of special stereotype (<<Unique>>, <<Index>>) -> method parameters define the indexed columns
Attribute propertiesAttribute properties, special stereotypes (see below)
ConstraintMethod of special stereotype <<Constraint>>
TriggerMethod of special stereotype <<Trigger>>
View Class of special stereotype <<View>> with special dependencies of stereotype <<Derive>>

Special SQL-attribute properties might be mapped this way:

SQLUML
NULL / NOT NULLIf an attribute should be NOT NULL, let the attribute be of stereotype <<NOT NULL>>. Else the attribute is nullable.
autoincstereotype <<AutoInc>>

The 'default value' and the 'enumeration' attribute properties are supported directly.
